🔐 Methods to Secure Data & Information

📌 Overview

There are several methods used to protect data from unauthorised access and cyber threats.

1

🔥 Firewall

Controls network traffic

Allows or blocks data based on security rules.

2

🔒 Encryption

Scrambles data using a key

Protects data so it cannot be read if intercepted.

3

🔑 Passwords & Biometrics

Authenticates users
  • Use strong passwords (long + complex)
  • Include symbols, numbers, upper/lowercase
  • Change regularly
💡 Exam Tip: Strong password = long + complex + unique
4

🤖 CAPTCHA & reCAPTCHA

Distinguishes humans from bots

CAPTCHA tests ensure that a real human is entering data, not an automated program (bot).

⚙️ How it works
  • User completes a task that is easy for humans but difficult for computers
  • Prevents bots from creating accounts or spamming systems
📊 Types
  • Typing distorted letters/numbers
  • Selecting images (e.g. traffic lights)
  • Audio CAPTCHA
  • reCAPTCHA: uses images or text from real-world data

reCAPTCHA also helps improve services by identifying objects in images and digitising text.

💡 Exam Tip:
CAPTCHA = stop bots,
reCAPTCHA = smarter version using real data
5

🛡️ Anti-Malware

Detects, blocks, and removes malware

Anti-malware software protects a computer by scanning files and monitoring system activity to detect threats.

⚙️ How it works
  • Scans files for known malware signatures (patterns)
  • Monitors behaviour to detect suspicious activity
  • Blocks or quarantines infected files
  • Allows users to run full system scans
🔄 Updates
  • Regular updates add new malware definitions
  • Keeps protection effective against new threats
6

📂 Access Rights

Controls file permissions

Limits what users can read, write or modify.

7

🔐 HTTPS

Secure version of HTTP

HTTPS (Hypertext Transfer Protocol Secure) is used to securely transfer data between a web browser and a web server.

  • Encrypts data during transmission
  • Prevents hackers from reading data
  • Used for secure websites (e.g. banking, shopping)
  • Shows a 🔒 padlock in the address bar
💡 Exam Tip: HTTPS = encrypted + secure communication
8

📧 Email Attachments & Web Links

Be cautious with files and links

Email attachments and web links can contain malware or lead to fake websites designed to steal personal information.

⚠️ Risks
  • Attachments may contain viruses or malware
  • Links may lead to phishing or fake websites
🛡️ Safety Tips
  • Do not open attachments from unknown senders
  • Hover over links to check the real URL
  • Avoid clicking suspicious or urgent messages
  • Be careful with .exe or .zip files
  • Check for spelling or grammar mistakes
💡 Exam Tip: Never trust unknown attachments or links
9

💾 Backup

A backup is a copy of data stored separately so it can be restored if lost or damaged.
i

📀 Full Backup

Copies all data
  • All files are backed up every time
  • Easy to restore
  • Takes the most time and storage
ii

🗂️ Incremental Backup

Backs up changes since last backup
  • Only new or changed files are saved
  • Fast and uses less storage
  • Restore can be slower (needs multiple backups)
iii

📁 Differential Backup

Backs up changes since last full backup
  • Stores all changes since last full backup
  • Faster restore than incremental
  • Uses more storage than incremental

💾 Good Backup Procedures

Best practices for protecting data
  • Set automatic backups
  • Avoid optical media (can be damaged easily)
  • Schedule backups when systems are not in use
  • Create more than one backup copy
  • Store backups in different locations
  • Use online/cloud storage
  • Keep important data in a fireproof safe
